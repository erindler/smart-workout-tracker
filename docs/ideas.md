# Application Ideas

 - Storing lifts in a way so that the type of weight isn't particular and that there is a natural grouping. Helpful for if all of one weight is taken during your lift, you can pivot and do something else. (i.e. all barbell benches are taken but dumbbells with benches aren't, you can swap barbell bench press with dumbbell bench press). One should also be able to compare the weights across the types. So can compare barbell weights side-by-side with dumbbell weights if desired. Maybe an algorithm to put them on the same scale too?
 - Be able to share workouts/programs. Would be good to have multiple methods of sharing and being able to copy a shared one to make it your own. Can also add access levels if collaboration is desired.
 - In lift, would like the number of sets and reps to be based on what's in the program's workout.
 - Before adding the built-in AI coach, can add an API endpoint for actions we would like an AI agent to take, such as building programs and implementing in the app. The main use case I am thinking of now is creating an API endpoint for creating a program. Currently when I add a program I ask LLM to create a program to solve a given goal and then I have to put it into my workout tracker. Would be cool if LLM could directly input it itself.
